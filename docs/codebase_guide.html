1. מה הבוט עושה — בשפה פשוטה

בוט WhatsApp שמנהל שיחות בעברית עם לידים של Propeller Drones — חברה שמוכרת רחפנים ומפעילה אקדמיה לטייסים.

הליד ממלא טופס בפרסומת ב-Facebook → הבוט מקבל אותו ב-WhatsApp → מנהל איתו שיחה → מנסה לסגור שיחה עם נציג מכירות.

שלבמה קורהמי עושה את זה
1ליד ממלא טופס בפרסומתFacebook Ads
2LeadMe (CRM) שולח webhook לבוטLeadMe CRM
3הבוט שולח הודעת פתיחה ב-WhatsAppopener.py
4הליד עונה → GPT-4o מנהל שיחהagent/graph.py
5הבוט מנסה לקבוע שיחה עם יועץכלי schedule_call
6פרטי הליד עוברים ל-LeadMe → נציג מתקשרcrm/leadme_client.py

2. זרימה: הודעה נכנסת מ-WhatsApp

כאשר ליד שולח הודעה, זה בדיוק מה שקורה בקוד:

📡
GreenAPI מקבל הודעה
ספריית whatsapp-chatbot-python מבצעת polling ל-GreenAPI בלולאה אינסופית. כשמגיעה הודעה — היא מגיעה כ-JSON.
app/main.py → bot.run_forever()
🔎
חילוץ פרטי השולח
מוציאים מהנוטיפיקציה: מספר טלפון, שם, תוכן ההודעה. בודקים שזו לא הודעת קבוצה. בודקים שהמספר ב-allow-list (אם מוגדר).
app/whatsapp/handler.py → _extract_text(), _extract_sender_info(), _phone_from_chat_id()
🔇
בדיקת Mute
אם ה-admin השתיק את הבוט לליד הזה (bot_muted=True) — ההודעה נשמרת ב-DB אבל הבוט לא עונה.
app/whatsapp/handler.py → lead.bot_muted
💾
טרנזקציה 1: שמירת ההודעה ל-DB
לפני שמריצים AI — שומרים את ההודעה ב-DB. כך אם ה-AI יתרסק, ההודעה לא תאבד. גם בודקים: האם זו ההודעה הראשונה של הליד? אם כן — מסמנים לדחוף "Level 2" ל-LeadMe.
app/agent/graph.py → handle_message() → Transaction 1
🤖
טרנזקציה 2: הרצת ה-AI Agent
בונים את ה-System Prompt (כולל מצב הליד הנוכחי + קטלוג הסרטונים). טוענים 30 ההודעות האחרונות כהיסטוריה. מריצים את LangGraph ReAct Agent שיכול להפעיל כלים.
app/agent/graph.py → Transaction 2 → _agent().invoke()
🔒
Safety Nets (רשתות ביטחון)
אחרי שה-AI ענה, מריצים בדיקות:
• אם התשובה באנגלית — מריצים את ה-AI שוב עם תזכורת לענות בעברית
• מנקים markdown links (WhatsApp לא מציג אותם)
• מנקים משפטי מילוי ("אני כאן לעזור") — הלקוח ביקש להסיר אותם
• אם ה-AI אמר "קבעתי שיחה" אבל לא קרא לכלי — קוראים לכלי אוטומטית
app/agent/graph.py → _strip_filler(), _strip_markdown_links(), _enforce_booking_promise()
📤
שליחת תשובה
מציגים "מקליד..." ← ממתינים כמה שניות (אנושי יותר) ← שולחים את התשובה דרך GreenAPI.
app/whatsapp/sender.py → ChatSender.send_text(humanize=True)

3. זרימה: ליד חדש מ-LeadMe

📨
LeadMe שולח Webhook
כשמישהו ממלא טופס בפרסומת — LeadMe שולח HTTP POST לכתובת /webhook/leadme/{secret} עם פרטי הליד.
app/webhook/server.py → leadme_webhook()
🔬
פרישת הנתונים
חולצים מהבקשה: טלפון (ומנרמלים לפורמט ישראלי), שם, שדות מותאמים אישית (clf_XXXXX), מזהה קמפיין.
app/webhook/server.py → _normalize_phone(), _extract_custom_fields()
⚡
תגובה מיידית 200 OK
הבוט מחזיר 200 מיד ← LeadMe לא מחכה. הטיפול בליד קורה ב-thread נפרד ברקע כדי לא לגרום timeout.
app/webhook/server.py → Thread(daemon=True).start()
🔍
בדיקה: כבר פנינו לליד?
מחפשים בDB אם הליד קיים ואם כבר נשלחה הודעת פתיחה (opener_sent_at). אם כן — לא שולחים שוב.
app/webhook/opener.py → handle_new_lead()
👋
שליחת הודעת פתיחה
בוחרים תבנית פתיחה (לפי שם הליד ולפי מזהה הקמפיין) → שולחים דרך GreenAPI → שומרים ב-DB עם timestamp.
app/webhook/opener.py → _render_opener(), _pick_topic()

4. זרימה: Follow-up אוטומטי

כל 30 דקות רץ job שמחפש לידים שלא ענו — ושולח להם הודעות תזכורת.

⏰
APScheduler מפעיל Tick
כל FOLLOWUP_INTERVAL_MINUTES (ברירת מחדל: 30 דקות) קורה run_once().
app/followup/scheduler.py → run_once()
🌙
בדיקת "שעות שקט"
אם השעה בישראל היא לפני 9:00 או אחרי 20:00 — לא שולחים כלום. גם שישי-שבת לא שולחים.
app/followup/scheduler.py → _is_within_quiet_hours()
🎥
Follow-up לוובינר
מחפש לידים שקיבלו את הוובינר לפני X שעות ועדיין לא ענו — שולח "ראית את הוובינר?"
app/followup/scheduler.py → _pick_webinar_followups()
🔕
מציאת לידים שקטים
מחפש לידים שעומדים בכל התנאים: לא handed_off, לא muted, לא קבעו שיחה, ההודעה האחרונה היא שלנו (הם לא ענו), עברו 24+ שעות.
app/followup/scheduler.py → _pick_leads_to_nudge()
📤
שליחת Nudge (דחיפה)
שולח הודעת תזכורת קבועה (לא AI — זול ומדויק). Nudge 1 = אחרי 24ש, Nudge 2 = אחרי 72ש. מקסימום 2 nudges.
app/followup/scheduler.py → _send_and_record(), _render_nudge()
📊
דיווח Level 3 ל-LeadMe
אם הליד לא ענה אפילו פעם אחת אחרי שני nudges — דוחפים "Level 3" ל-LeadMe ("לא הגיב").
app/crm/client.py → mark_no_reply()

7. הכלים של ה-AI

ה-AI (GPT-4o) יכול להפעיל 6 כלים בזמן שיחה. כל כלי הוא פונקציית Python שה-AI קורא לה:

🔍 search_knowledge(query, topic?)
חיפוש סמנטי ב-ChromaDB — מאגר הידע שנבנה מגריידת האתר ומסמכים מקומיים. מחזיר קטעי טקסט רלוונטיים.
מתי: בכל שאלה עובדתית (מחיר, רישיון, מסלולים, מיקומים)
🏷️ classify_lead(familiarity?, stage?, intent?, ...)
עדכון מצב הליד ב-DB. כל הפרמטרים אופציונליים — רק מה שהתחדש. שומר: familiarity, funnel stage, כוונה, תעשייה, חלון שעות.
מתי: בכל פעם שמתגלה מידע חדש על הליד
🎬 send_video(video_id, caption?)
שולח וידאו ל-WhatsApp. מונע שליחה כפולה (בודק videos_sent + in-turn dedup). אם הוובינר נשלח — שומר webinar_sent_at לצורך follow-up.
מתי: כשמנסים לחמם ליד עם תוכן חזותי
💡 recommend_video(topics_context?)
מקבל המלצה לוידאו הבא לפי familiarity של הליד, נושאים שדובר עליהם, וסרטונים שכבר נשלחו. מחזיר רק את ה-id לשימוש ב-send_video.
מתי: לפני send_video כשלא בטוחים איזה וידאו
📅 schedule_call(summary?, preferred_call_slot?)
מסמן את הליד כ-"handed_off", שולח ל-LeadMe, מעדכן funnel_stage. דורש חלון שעות חוקי (9-12 / 12-15 / 15-18 / any). אם אין — מחזיר הנחיה לשאול.
מתי: רק אחרי שהליד אמר מפורשות שרוצה שיחה ונתן חלון
❌ cancel_call(reason?)
מבטל שיחה שכבר תואמה. מנקה preferred_call_slot ומחזיר funnel_stage ל-"warm". שולח ביטול ל-LeadMe.
מתי: רק אם הליד ביקש לבטל/לשנות שיחה שתואמה
חשוב להבין: ה-AI לא "שולח" הודעות ישירות — הוא קורא לכלים, והכלים עושים את העבודה. הכלי send_video הוא זה שמתקשר ל-GreenAPI, לא ה-AI עצמו.

8. Safety Nets — רשתות ביטחון

הבוט סובל מבעיות ידועות של LLM — כל אחת קיבלה "safety net" שמתקן אותה אחרי שה-AI עונה:

🌐 Hebrew Safety Net אם התשובה מכילה פחות מ-15% תווים עבריים — מריצים שוב עם הנחיה לענות בעברית. אם גם זה לא עוזר — שולחים תשובת ברירת מחדל קבועה.

קובץ: graph.py → _looks_like_english()
✂️ Filler Stripper מנקה שורות מסוג "אני כאן לעזור", "מוזמן לפנות", "מקווה שעזרתי" — הלקוח ביקש להסיר אותן מפורשות. פועל על כל שורה אחרונה בנפרד.

קובץ: graph.py → _strip_filler()
🔗 Markdown Link Cleaner הופך [טקסט](url) ל-URL בלבד, ומסיר **url**. WhatsApp לא מציג markdown — הליד היה רואה סוגריים גלויים.

קובץ: graph.py → _strip_markdown_links()
📋 Booking Promise Enforcer אם ה-AI אמר "קבעתי לך שיחה" אבל schedule_call לא רץ — קורא ל-mark_ready_for_call() אוטומטית. מונע הבטחה ריקה.

קובץ: graph.py → _enforce_booking_promise()
שים לב לבאגים נפוצים: אם הבוט שולח תשובות באנגלית → בדוק את _looks_like_english(). אם הבוט שולח "אני כאן לעזור" → בדוק שה-filler patterns ב-graph.py מכסים את הוריאציה. אם הבוט קבע שיחה בלי שהליד ביקש → בדוק את slot validation ב-classify_lead.

11. רמות Engagement (ל-LeadMe)

במקביל למשפך הפנימי, דוחפים ל-LeadMe סטטוס engagement:

13. מה ללמוד — לפי סדר עדיפויות

1
Python בסיסי
משתנים, פונקציות (def), if/else, לולאות, קלאסים בסיסיים, import. זמן: 3-5 ימים.
אם יש לך JS: Python זה JS בלי סוגריים. במקום function → def. במקום var/const → אין כלום. Indentation חשוב!
2
HTTP ו-REST API
מה זה GET/POST, headers, JSON body, status codes. כנראה יש לך את זה מ-React (fetch/axios). כאן משתמשים ב-httpx ו-requests.
זהה ל-fetch() ב-JS. requests.post(url, json={...}) = fetch(url, {method:'POST', body:...})
3
FastAPI
הספרייה שמגדירה HTTP endpoints (routes). ב-app/webhook/server.py ו-app/admin/routes.py. מאוד דומה ל-Express.js.
@app.post("/webhook") = app.post('/webhook', (req,res) => {}). Pydantic = Zod/TypeScript types
4
SQLAlchemy (ORM)
איך להגדיר טבלאות (models.py) ואיך לשלוף/להכניס נתונים (repository.py). משתמשים ב-session_scope() לניהול transactions.
SQLAlchemy ≈ Prisma/Drizzle. session.get(Lead, id) = prisma.lead.findUnique({where:{id}})
5
LangChain + LangGraph
הספרייה שמחברת GPT לכלים. create_react_agent() יוצר agent שיכול לקרוא לפונקציות. כל @tool הוא פונקציה שה-AI יכול "לבחור" להפעיל.
אין אנלוגיה ב-JS נפוצה, אבל הלוגיקה: AI מקבל רשימת כלים → מחליט מה לקרוא → מריץ → מחזיר תוצאה
6
Docker ו-Docker Compose
הפרויקט רץ ב-3 containers: postgres, chroma, bot. docker-compose up מפעיל הכל. docker logs -f bot לראות לוגים.
אין אנלוגיה. Docker = "קופסה" שבתוכה רץ הקוד עם כל התלויות, בלי להתקין כלום על המחשב ישירות.
7
Context Managers (with)
Python feature שמשמשת הרבה כאן: with session_scope() as session: — פותח transaction, מבצע קוד, סוגר אוטומטית (כמו try/finally).
דומה ל-try { db.begin(); ... await db.commit() } catch { await db.rollback() } finally { db.close() }
טיפ: לטפל בבאגים לא צריך לדעת הכל. בדרך כלל: קרא את שגיאת הLOG, מצא את שם הקובץ והשורה, פתח את הקובץ, הבן את הפונקציה הספציפית. 90% מהבאגים ניתנים לתיקון גם בלי להבין את כל הקוד.
